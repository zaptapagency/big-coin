Architecture:
──────────────
  Browser → HTTP → NodeRPCServer → BigCoinFullNode
                                       ↓
                                   P2PNode ←→ Other Peers
                                       ↓
                                  Blockchain (local)


How It Works:
──────────────
  1. Each user runs a full node (p2p_node_rpc_server.py)

  2. Nodes connect to each other via TCP P2P network

  3. When someone mines a block:
     - Node validates it
     - Broadcasts to all peers
     - Other nodes receive and sync
     - All have same blockchain!

  4. Client connects via HTTP to RPC server
     - Wallet operations
     - Mining control
     - Blockchain queries


================================================================================
                          KEY DIFFERENCES
================================================================================

Old System (Centralized):
  Client1 ─┐
  Client2 ─├─→ shared_state.py (single server)
  Client3 ─┘

  ✗ Single point of failure
  ✗ Server must stay online
  ✗ Can't run offline


New System (Decentralized):
  Node1 ←→ Node2 ←→ Node3
   ↑       ↑        ↑
  Each is complete blockchain

  ✓ No single point of failure
  ✓ Works even if nodes go down
  ✓ Can run offline (and catch up later)
  ✓ Real peer-to-peer network

================================================================================
                        TROUBLESHOOTING
================================================================================

Problem: "Address already in use"
Solution:
  taskkill /F /IM python.exe
  Then start fresh

Problem: Nodes not connected
Solution:
  python connect_nodes.py
  (This connects all nodes together)

Problem: Blocks not syncing
Solution:
  Wait 30 seconds for background sync
  Or check: curl http://localhost:8002/api/blockchain/info

Problem: Mining is slow
Solution:
  That's normal! Mining takes time (10-60 seconds)
  Difficulty can be lowered in params.py for testing

================================================================================
                           IMPORTANT NOTES
================================================================================

1. This is peer-to-peer but running locally on one machine
   - Alice, Bob, Charlie are separate Python processes
   - In production, they'd run on different servers/locations

2. All three nodes share same genesis block
   - They start synchronized
   - Mining keeps them synchronized

3. Can add/remove nodes anytime
   - Each node is independent
   - New nodes catch up automatically

4. For production:
   - Run on real servers (AWS, DigitalOcean, etc.)
   - Use real IP addresses (not localhost)
   - Configure firewalls
   - Add monitoring

5. This is educational only
   - Not for real money
   - Not production-grade security
   - For learning P2P concepts
