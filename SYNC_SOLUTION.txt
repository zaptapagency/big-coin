================================================================================
                    MYCOIN — SYNCHRONIZED UI SOLUTION
================================================================================

PROBLEM FIXED:
  - Browser and Desktop GUI had separate blockchains
  - Addresses didn't match between apps
  - Mining on Desktop didn't appear on Web
  - All state was duplicated and out-of-sync

================================================================================
                            SOLUTION: IPC SERVER
================================================================================

Created a Shared State Server that all UIs connect to:

  ┌─────────────────────────────────────┐
  │  shared_state.py (port 9999)        │
  │                                     │
  │  Holds: Single Node instance       │
  │  Tracks: Addresses, mining state   │
  │  Serves: JSON RPC over TCP         │
  └──────────────┬──────────────────────┘
                 │
     ┌───────────┼──────────────┐
     |           |              |
  [Web]      [Desktop]      [Mobile]
  (5000)       GUI         (Flutter)

All 3 UIs now use the SAME blockchain!

================================================================================
                         FILES CREATED (4 NEW)
================================================================================

1. shared_state.py
   - TCP server on port 9999
   - Manages Node instance (shared.py)
   - Handles addresses, mining, blockchain queries
   - Thread-safe state management
   - JSON RPC interface

2. shared_client.py
   - Client library for web_app & gui
   - Methods: new_key(), get_balance(), blockchain_info(),
             start_mining(), mining_status(), stop_mining()
   - Auto-reconnect on socket errors

3. web_app_shared.py
   - Web dashboard that uses shared_client
   - Flask server on port 5000
   - Same routes as web_app.py but queries shared state
   - All UIs see same addresses & blockchain

4. gui_shared.py
   - Desktop GUI that uses shared_client
   - PyQt6 window with 4 tabs
   - Connects to shared_state.py on startup
   - Threading for non-blocking mining

================================================================================
                          API REFERENCE
================================================================================

from shared_client import SharedStateClient

client = SharedStateClient("127.0.0.1", 9999)

# Generate new keypair
result = client.new_key()
# Returns: {"address": "1ABC...", "pubkey_hash": "def..."}

# Get balance
result = client.get_balance()
# Returns: {"balance_coins": 150.0, "balance_cents": 15000000000}

# Start mining
result = client.start_mining(blocks=5, address="1ABC...")
# Returns: {"status": "mining started"}

# Check mining progress
result = client.mining_status()
# Returns: {"is_mining": true, "blocks_mined": 2, "blocks_to_mine": 5, ...}

# Get blockchain info
result = client.blockchain_info()
# Returns: {"height": 5, "tip_hash": "0000abc...", "total_money_coins": 250.0, ...}

# Stop mining
result = client.stop_mining()
# Returns: {"status": "mining stopped"}

# Get all addresses
result = client.get_addresses()
# Returns: {"addresses": [{"address": "1ABC...", "pubkey_hash": "def..."}, ...]}

================================================================================
                         WHAT CHANGED
================================================================================

ORIGINAL (web_app.py, gui.py):
  - Each created its own Node("web-app") or Node("gui-app")
  - Separate blockchains, addresses, state
  - Mining on one didn't affect the other

UPDATED (web_app_shared.py, gui_shared.py):
  - Both import SharedStateClient
  - Both connect to shared_state.py (port 9999)
  - All queries go through shared_client
  - Same blockchain, addresses, mining state

shared_state.py (NEW):
  - Single Node("shared", coinbase_maturity=0) instance
  - Thread-safe: state_lock for concurrent access
  - Serves JSON RPC requests over TCP

================================================================================
                         TROUBLESHOOTING
================================================================================

Q: "Connection refused" error in web_app_shared.py?
A: Make sure shared_state.py is running first (Terminal 1)

Q: Desktop GUI crashes on startup?
A: Run shared_state.py first, then gui_shared.py
   Also check that port 9999 is not blocked

Q: Addresses still don't match?
A: Make sure you're using web_app_SHARED.py and gui_SHARED.py
   (Not the original web_app.py and gui.py)

Q: Mining on Desktop doesn't update Web?
A: Refresh the Web Dashboard page (it polls /api/blockchain/info)
   Or wait 3 seconds for auto-refresh

Q: Can I still use original web_app.py and gui.py?
A: Yes, they still work! They just don't share state.
   Use *_shared.py versions for synchronization.
